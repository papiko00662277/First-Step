# FirstStep ― 最初の一歩フォーカスアプリ

曖昧で大きいタスクを、**2分で着手できる「最初の一歩」**に変換し、そのまま “今やること1つ＋タイマー” の集中画面でこなしていくための、スマホ向け単一ページアプリです。先延ばし（タスク着手のハードル）を下げることを目的にしています。

生成AI活用基礎（大学院課題）で、要件定義から設計・実装まで行った個人用ツールです。

## できること

- **AIがタスクを手順に分解**：入力したタスクを、実行順の4〜6ステップに分解。最初の一歩は「動詞＋具体物・2分以内で着手可能」に限定。
- **フォーカス実行画面**：今やること1つだけを全画面表示＋カウントダウン。こなすと次の一歩を提示（着手の勢いを継続へ）。
- **強制ペナルティ（ロックアウト）**：スキップ／中断するとアプリが一定時間ロックして次に進めなくする自己制御機能。スキップを重ねると延長（上限あり）。
- **ストリーク**：連続達成をブラウザに記録。スキップで途切れる（損失回避）。
- **2つの使い方**：
  - ①**APIを使う**：自分のAPIキーでブラウザから直接呼び出し、上記フォーカス実行まで自動。
  - ②**APIなし**：キー不要。工夫を組み込んだプロンプトを生成してコピー→手元のChatGPT/Claudeに貼る。

## 使い方（iPhone / スマホ）

1. 公開URLを Safari で開く。
2. （AIモードを使う場合）`aistudio.google.com` で **無料のGemini APIキー** を取得し、接続設定に貼る。無料枠は目安で1日約1,500回まで。
3. タスクを入力 →「AIに手順を作ってもらう」→ 集中画面で進める。キー不要で使いたいときは「APIなしでプロンプトを作る」。
4. Safari の共有 →「ホーム画面に追加」で全画面アプリとして起動できます。

> プライバシー：APIキーはブラウザのメモリ内だけで使い、送信先は選んだAIの公式APIのみ。ページのソースにキーは含まれません。無料枠は入力がモデル改善に使われ得るため、機微な情報は書かないでください。

## 設計の工夫（プロンプト）

単純に「分解して」と頼むと抽象的な手順になりがちなので、次の工夫をプロンプトに組み込んでいます。

- 役割付与＋粒度制約（動詞＋具体物・最初は2分・抽象語禁止）
- 出力前の自己検証（最初の一歩は2分で着手可能か点検・修正）
- 状況適応（締切・状態・つまずきを反映）
- 穏当な自己ペナルティの後押し
- 出力はJSONで受け取り、アプリがフォーカスUI＋タイマーへ変換

## 根拠（すべて CORE Rank A 以上の会議論文）

- Cheng et al. (2015, CHI). *Break it down: A comparison of macro- and microtasks.* — タスク分解で質・中断耐性が向上。
- Arakawa, Yakura & Goto (2023, CHI). *CatAlyst.* — 生成AIで停滞を検知し着手/再開のハードルを下げる。
- Lee et al. (2021, CHI). *Sticky Goals.* — 金銭的コミットメントで達成率が向上。
- Kim et al. (2019, IMWUT/UbiComp). *GoalKeeper.* — 自分で課すロックアウト（ペナルティ）が自己制御に有効。

## 差別化

既存アプリは「AIでタスク分解する着手支援系（Thawly, ADHD Guardian 等）」か「お金を賭けるペナルティ系（stickK, Beeminder, Forfeit）」のどちらかに分かれる。本アプリは両者を統合し、**AIが切り出した“2分の最初の一歩”に、アプリ内ロックアウトという強制可能なペナルティをかける**点が独自。

## 技術メモ

- 依存ライブラリなしの単一HTML（`index.html`）。GitHub Pages でそのまま公開可能。
- OpenAI互換エンドポイント（Google Gemini / Groq / OpenRouter / OpenAI）と Anthropic に対応。
- ストリークは `localStorage`、キーはメモリ内のみ。

## セキュリティ（公開リポジトリでも安全な理由）

このリポジトリは公開(public)ですが、秘密情報は含みません。

- **APIキーはソースに含まれません。** キーは利用者がブラウザの入力欄に入れた瞬間だけ読み取られ、選んだAIの公式APIにのみ送信されます。
- **キーは保存もされません。** `localStorage` に保存しているのは連続達成カウント（整数）だけで、APIキーはメモリ内のみ・ページを閉じると消えます。
- キー入力欄は `type="password"` ・`autocomplete="off"`。
- したがってソースが公開されても、キーやタスク内容が第三者に渡ることはありません。

運用上の注意：キーを直接ソースに書き込んでコミットしない／URLは必要な範囲でのみ共有してください。無料APIの無料枠は入力がモデル改善に使われ得るため、タスク文に機微な情報は書かないでください。
